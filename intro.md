Today I'd like to present a project that is the sum of all my work after nearly a decade of BFME modding: **[pySAGE](https://github.com/ClementJ18/pySAGE)**. **pySAGE** is not, unlike what its name could suggest, a re-implementation of the SAGE engine in Python. Instead it is a monorepo containing cleaned-up and "productionized" versions of many of the applications I developed over my years of working on the Edain mod. These tools can be used by players, mod developers and tool developers to make their lives easier when it comes to the SAGE engine and its games. Two quick disclaimers before I dig in: the code linked has been written with AI assistance, and my main game is Rise of the Witch-king, so that is the game I test against. Other games should work but may need some changes to the code.

So, what is pySAGE? At its core pySAGE is a Python parser for SAGE ini files. It offers complete coverage, converting the ini code into Python objects which can then be validated and manipulated. Heavily inspired by the work of the [OpenSAGE](https://opensage.github.io/) team, it allows Python applications to "read" the code of SAGE games. Paired with [pyBIG](https://github.com/ClementJ18/pyBIG) you can also extract this code from the BIG archives the games usually come packaged in. This foundation then allowed me to develop sage_lint, a linter - an automated tool for detecting errors in code without executing it - that is much stricter than the SAGE engine itself. Many modders will know the ungodly number of errors that the SAGE engine is willing to overlook just to crash during a match. This tool's strict linting will surface those errors, whether you misspelled an object, used an invalid KindOf, removed a module that doesn't exist or assigned an object to multiple ExperienceLevels. This greatly speeds up your debugging, as you are alerted to errors without having to launch the game, and it even offers helpful advice on code smells - code that isn't necessarily an error but would be considered bad practice. This tool is made available to non-technical users through the SAGE Lint UI or the SAGE Lint Sublime Text 4 plugin. The linter also comes with a formatter that aligns your code on a common indentation, which makes it much cleaner to look at and allows collapsing certain modules. Finally, the core parser also ships a git merge tool that allows git to better understand BFME game code and do a better job of merging it.

On top of this core I also built the following tools and libraries:
- **BfMe Searcher (sage_ui)**: Inspired by [Prinz's Edain Searcher](https://edain.fandom.com/wiki/Edain_Searcher), this tool allows players and other non-technical users to look up actual game stats without needing other people to fetch them. A way to navigate the game without having to launch it. Useful for balance discussions and updating wikis.
- **Edain Wiki Assistant (sage_wiki)**: Very specific to Edain, but offers automation for keeping wikis up to date and in sync with the mod.
- **sage_replay** can parse BFME's replay files and, when paired with a game installation, translate them so that they can be analysed without needing the exact game version they were played in. This opens up a lot of potential for data analysis, as can be seen in some of the work I did [here](https://edain-mod.fra1.cdn.digitaloceanspaces.com/aggregate/edain-4.8.4.3/index.html).
- **sage_save** can parse BFME's save files. Still very WIP; the goal is to be able to understand and perhaps manipulate this format.
- **sage_map** can parse BFME's map files, transforming them into Python structures, and can also save them to JSON. They can be manipulated and recompiled to do automated fixes or fun alterations such as scaling. sage_map also includes a framework for linting but no base rules, as mapping rules are very mod-specific. In the sage_mods folder you can find examples of such rules created specifically for Edain.
- **sage_apt** is a very WIP editor for APT files; it is not yet fully functional and remains largely untested.

Many of these have dedicated .exes that can be used to either open a UI or run from the command line. All of these can be extended with a little Python knowledge to fine tune them to your needs. I hope that these tools can be used by modders to work more efficiently on BFME code. If you'd like to know more, each project in this monorepo has a detailed README including installation and download links.
